# Schema Reference

All EventFabric tables live in the `eventfabric` PostgreSQL schema. This
document provides a complete column-level reference for every table, index, and
constraint created by the migration files in `packages/postgres/migrations/`.

## Schema creation

```sql
CREATE SCHEMA IF NOT EXISTS eventfabric;
```

Run this before any migration. Migration `001_init.sql` includes this statement.

## Migration file listing

| File                            | Creates                                                        |
| ------------------------------- | -------------------------------------------------------------- |
| `001_init.sql`                  | `eventfabric` schema, `eventfabric.events` table, 2 indexes   |
| `002_projection_checkpoints.sql`| `eventfabric.projection_checkpoints` table                     |
| `003_outbox_and_dlq.sql`        | `eventfabric.outbox` table (3 indexes), `eventfabric.outbox_dead_letters` table (1 index) |
| `004_snapshots.sql`             | `eventfabric.snapshots` table, 1 index                         |

All migrations are idempotent (`CREATE TABLE IF NOT EXISTS`, `CREATE INDEX IF
NOT EXISTS`). Apply them in numeric order:

```bash
psql "$DATABASE_URL" -f packages/postgres/migrations/001_init.sql
psql "$DATABASE_URL" -f packages/postgres/migrations/002_projection_checkpoints.sql
psql "$DATABASE_URL" -f packages/postgres/migrations/003_outbox_and_dlq.sql
psql "$DATABASE_URL" -f packages/postgres/migrations/004_snapshots.sql
```

## Table: eventfabric.events

The append-only event log. Every domain event ever recorded lives here.

### Columns

| Column              | Type          | Nullable | Default   | Description                                                       |
| ------------------- | ------------- | -------- | --------- | ----------------------------------------------------------------- |
| `global_position`   | `BIGSERIAL`   | NOT NULL | auto-inc  | **PRIMARY KEY.** Monotonically increasing position across all streams. Used for ordered global reads by catch-up and async projections. |
| `event_id`          | `UUID`        | NOT NULL | --        | Unique identifier for the event, generated by the application (`randomUUID()`). |
| `aggregate_name`    | `TEXT`        | NOT NULL | --        | The static `aggregateName` of the aggregate class (e.g. `"Account"`). Identifies the stream type. |
| `aggregate_id`      | `TEXT`        | NOT NULL | --        | The identity of the aggregate instance (e.g. `"acc-1"`). Together with `aggregate_name`, identifies the stream. |
| `aggregate_version` | `INT`         | NOT NULL | --        | Sequential version within the stream, starting at `1`. Used for optimistic concurrency. |
| `type`              | `TEXT`        | NOT NULL | --        | The event type discriminator (e.g. `"AccountDeposited"`). Matches `payload.type`. |
| `version`           | `INT`         | NOT NULL | --        | The event schema version (e.g. `1`). Matches `payload.version`. Stored as a top-level column for efficient filtering. |
| `payload`           | `JSONB`       | NOT NULL | --        | The full event object serialized as JSON. Contains `type`, `version`, and all domain-specific fields. |
| `occurred_at`       | `TIMESTAMPTZ` | NOT NULL | `now()`   | Timestamp of when the event was appended. Set by the database, not the application. |
| `dismissed_at`      | `TIMESTAMPTZ` | NULL     | `NULL`    | If non-null, the event has been soft-deleted. Set by `PgEventStore.dismiss()`. |
| `dismissed_reason`  | `TEXT`        | NULL     | `NULL`    | Optional human-readable reason for dismissal (e.g. `"GDPR erasure"`). |
| `dismissed_by`      | `TEXT`        | NULL     | `NULL`    | Optional identifier of the actor who dismissed the event. |
| `correlation_id`    | `TEXT`        | NULL     | `NULL`    | Optional tracing correlation ID. Passed via `meta.correlationId` at append time. |
| `causation_id`      | `TEXT`        | NULL     | `NULL`    | Optional tracing causation ID. Passed via `meta.causationId` at append time. |

### Constraints

| Constraint                                              | Type       | Description                                                     |
| ------------------------------------------------------- | ---------- | --------------------------------------------------------------- |
| `events_pkey` (implicit)                                | PRIMARY KEY | `(global_position)`. Guarantees global ordering.               |
| `events_event_id_key` (implicit)                        | UNIQUE     | `(event_id)`. Prevents duplicate event IDs.                    |
| `events_aggregate_name_aggregate_id_aggregate_version_key` | UNIQUE | `(aggregate_name, aggregate_id, aggregate_version)`. Enforces optimistic concurrency -- two events cannot claim the same version slot in the same stream. |

### Indexes

| Index name          | Columns                                                    | Purpose                                                          |
| ------------------- | ---------------------------------------------------------- | ---------------------------------------------------------------- |
| `events_stream_idx` | `(aggregate_name, aggregate_id, aggregate_version)`        | Fast stream loads (`loadStream`). Covers the common query `WHERE aggregate_name = $1 AND aggregate_id = $2 AND aggregate_version >= $3 ORDER BY aggregate_version ASC`. |
| `events_global_idx` | `(global_position)`                                        | Fast global position reads (`loadGlobal`). Covers the query `WHERE global_position > $1 ORDER BY global_position ASC LIMIT $2`. |

## Table: eventfabric.projection_checkpoints

Tracks the last processed `global_position` for each catch-up or async
projection. One row per projection name.

### Columns

| Column                  | Type          | Nullable | Default  | Description                                                     |
| ----------------------- | ------------- | -------- | -------- | --------------------------------------------------------------- |
| `projection_name`       | `TEXT`        | NOT NULL | --       | **PRIMARY KEY.** Unique name of the projection (e.g. `"deposit-audit"`, `"email-notification"`). |
| `last_global_position`  | `BIGINT`     | NOT NULL | `0`      | The `global_position` of the last event successfully processed by this projection. The projection reads events with `global_position > last_global_position`. |
| `updated_at`            | `TIMESTAMPTZ` | NOT NULL | `now()`  | Timestamp of the last checkpoint update. Useful for monitoring staleness. |

### Constraints

| Constraint                        | Type        | Description                    |
| --------------------------------- | ----------- | ------------------------------ |
| `projection_checkpoints_pkey`     | PRIMARY KEY | `(projection_name)`.           |

## Table: eventfabric.outbox

The transactional outbox. When events are appended with `enqueueOutbox: true`,
a row per event is inserted here inside the same transaction. Async projection
runners claim rows, process them, and delete them on success.

### Columns

| Column              | Type          | Nullable | Default  | Description                                                       |
| ------------------- | ------------- | -------- | -------- | ----------------------------------------------------------------- |
| `id`                | `BIGSERIAL`   | NOT NULL | auto-inc | **PRIMARY KEY.** Internal row identity.                           |
| `global_position`   | `BIGINT`      | NOT NULL | --       | **UNIQUE.** References the `global_position` of the event in `eventfabric.events`. Used to load the full event payload when processing. |
| `topic`             | `TEXT`        | NULL     | `NULL`   | Optional topic string for message routing. Async runners can filter by topic when claiming batches. |
| `created_at`        | `TIMESTAMPTZ` | NOT NULL | `now()`  | Timestamp when the outbox row was created (same as the event append). |
| `locked_at`         | `TIMESTAMPTZ` | NULL     | `NULL`   | Set when a worker claims this row. Rows with `locked_at IS NOT NULL` are being processed and are not available for claiming. |
| `locked_by`         | `TEXT`        | NULL     | `NULL`   | Identifier of the worker that claimed this row (the `workerId` from `AsyncRunnerOptions`). |
| `attempts`          | `INT`         | NOT NULL | `0`      | Number of processing attempts. Incremented on each `releaseWithError`. Used to determine when to dead-letter the message. |
| `last_error`        | `TEXT`        | NULL     | `NULL`   | The error message from the most recent failed processing attempt. |
| `dead_lettered_at`  | `TIMESTAMPTZ` | NULL     | `NULL`   | Set when the message is moved to the dead letter queue. Rows with `dead_lettered_at IS NOT NULL` are no longer active. |
| `dead_letter_reason`| `TEXT`        | NULL     | `NULL`   | Reason the message was dead-lettered (e.g. `"Exceeded maxAttempts=5"`). |

### Constraints

| Constraint                     | Type        | Description                         |
| ------------------------------ | ----------- | ----------------------------------- |
| `outbox_pkey` (implicit)       | PRIMARY KEY | `(id)`.                             |
| `outbox_global_position_key`   | UNIQUE      | `(global_position)`. One outbox row per event. |

### Indexes

| Index name          | Columns                                       | Purpose                                                    |
| ------------------- | --------------------------------------------- | ---------------------------------------------------------- |
| `outbox_ready_idx`  | `(dead_lettered_at, locked_at, id)`            | Claiming unprocessed rows. The `claimBatch` query filters `WHERE dead_lettered_at IS NULL AND locked_at IS NULL ORDER BY id LIMIT $1`. |
| `outbox_topic_idx`  | `(topic)`                                      | Filtering by topic when claiming with a topic constraint.  |
| `outbox_global_idx` | `(global_position)`                            | Looking up the event for a given outbox row.               |

## Table: eventfabric.outbox_dead_letters

Messages that exceeded `maxAttempts` or could not be resolved are moved here
from `eventfabric.outbox`. This table is append-only and serves as an audit
trail for failed processing.

### Columns

| Column              | Type          | Nullable | Default  | Description                                                     |
| ------------------- | ------------- | -------- | -------- | --------------------------------------------------------------- |
| `id`                | `BIGSERIAL`   | NOT NULL | auto-inc | **PRIMARY KEY.** Internal row identity.                         |
| `outbox_id`         | `BIGINT`      | NOT NULL | --       | The `id` of the original row in `eventfabric.outbox`.           |
| `global_position`   | `BIGINT`      | NOT NULL | --       | The `global_position` of the event. Used to look up the original event for reprocessing or investigation. |
| `topic`             | `TEXT`        | NULL     | `NULL`   | The topic of the original outbox row.                           |
| `attempts`          | `INT`         | NOT NULL | --       | Number of processing attempts before dead-lettering.            |
| `last_error`        | `TEXT`        | NULL     | `NULL`   | The error message from the final failed attempt.                |
| `dead_lettered_at`  | `TIMESTAMPTZ` | NOT NULL | `now()`  | Timestamp when the message was dead-lettered.                   |

### Constraints

| Constraint                           | Type        | Description           |
| ------------------------------------ | ----------- | --------------------- |
| `outbox_dead_letters_pkey` (implicit)| PRIMARY KEY | `(id)`.               |

### Indexes

| Index name              | Columns              | Purpose                                              |
| ----------------------- | -------------------- | ---------------------------------------------------- |
| `outbox_dlq_global_idx` | `(global_position)`  | Looking up dead-lettered messages by event position.  |

## Table: eventfabric.snapshots

Caches aggregate state at a specific stream version to accelerate aggregate
loading. One row per `(aggregate_name, aggregate_id)` pair; newer snapshots
overwrite older ones via the version-gated upsert in `PgSnapshotStore.save()`.

### Columns

| Column                    | Type          | Nullable | Default  | Description                                                     |
| ------------------------- | ------------- | -------- | -------- | --------------------------------------------------------------- |
| `aggregate_name`          | `TEXT`        | NOT NULL | --       | The static `aggregateName` of the aggregate class. Part of the composite primary key. |
| `aggregate_id`            | `TEXT`        | NOT NULL | --       | The identity of the aggregate instance. Part of the composite primary key. |
| `aggregate_version`       | `INT`         | NOT NULL | --       | The stream version at which this snapshot was taken. Only events with `aggregate_version > this` need to be replayed. |
| `created_at`              | `TIMESTAMPTZ` | NOT NULL | `now()`  | Timestamp when the snapshot was saved.                          |
| `snapshot_schema_version` | `INT`         | NOT NULL | --       | Schema version of the serialized state. Used by `PgSnapshotStore` to decide whether to upcast the `state` column on load. |
| `state`                   | `JSONB`       | NOT NULL | --       | The serialized aggregate state at the time of the snapshot.     |

### Constraints

| Constraint                 | Type        | Description                                     |
| -------------------------- | ----------- | ----------------------------------------------- |
| `snapshots_pkey` (implicit)| PRIMARY KEY | `(aggregate_name, aggregate_id)`. One snapshot per aggregate instance. |

### Indexes

| Index name              | Columns                                               | Purpose                                              |
| ----------------------- | ----------------------------------------------------- | ---------------------------------------------------- |
| `snapshots_version_idx` | `(aggregate_name, aggregate_id, aggregate_version)`   | Fast version lookups and the version-gated upsert in `PgSnapshotStore.save()`. |
